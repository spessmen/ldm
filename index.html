About

LDM is a sort of unofficial proto-standard library for DM. This is meant to document it, along with how we interact with the framework we currently have to make do with: the BYOND pre-processor, engine, and compiler.

As an added "bonus", we're also going to try to fill in the gaps in the official language reference.

And there's a lot of gaps. But maybe we can make developing in DM suck less.
 

Goals

  • Laying the groundwork for an open-source DM and BYOND. It's going to happen at some point, might as well be ready for it.

  • Writing a robust library of commonly used shit. See Greenspun's 10th rule, if we're going to reinvent the wheel in collaborative megaprojects we might as well do it right.

  • Putting together a library of knowledge about developing in DM and BYOND. The official language specification is woefully incomplete, and development by trial-and-error/oral tradition is ridiculous in 2014.
 

Principles

  • Minimalism. Everything in here should have a well-thought-out rationale, and "someone might want it" does not count. We should also prefer simple solutions to complex ones unless they provide a large enough benefit to make them worthwhile.

  • Reliability. A standard library should have solutions you can count on. It's nice if they're the fastest or the most powerful, but they must be robust.

  • Flexibility. Every setting should be an option. If feasible, they should be run-time options. Even debug capabilities and standards enforcement should be optional: if you want to turn them off, go ahead. It's your funeral. Basically, this code should be plug-and-play, period: it should be usable in every DM project with minor tweaks.

  • Performance. One of the shittiest parts of coding is having to make tradeoffs between clarity, readability, maintainability, simplicity, etc and performance. We should strive for solutions that improve both, and never whitewash shitty compromises.
